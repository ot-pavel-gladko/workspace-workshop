# aila-SDK Workspace Constraints

Guardrails for auditing workspaces generated by the Artisyn SDK (`workspace.py`
+ `aila_catalog_schema.Workspace`). Missing these rules leads to proposed
rewrites that get clobbered on the next `workspace.py generate` run.

## Detecting an aila-SDK workspace

Look for any of these markers (from workspace root):

- `workspace.py` at root and imports from `aila_catalog_schema`
- `.aila/` directory
- `WORKSPACE_BOOTSTRAP.md` at root
- `WORKSPACE_SELFTEST.md` at root
- `CLAUDE.md` header contains `Auto-generated by workspace.py. Do not edit.`
- `.claude/rules/delegation.md` header contains
  `**Auto-generated by workspace.py. Do not edit.**`

If at least two markers match → treat as aila-SDK workspace and apply
constraints below.

## Files that MUST NOT be rewritten

In aila-SDK workspaces, these are auto-generated artifacts. The skill MUST
propose rewrites to source files instead.

| Generated artifact | Source of truth | Regeneration command |
|---|---|---|
| `CLAUDE.md` | `workspace.py` + `prompts/*.md` | `python3 workspace.py generate --agent claude` |
| `.claude/agents/*.md` | `workspace.py` (Agent(...) declarations) + `prompts/<agent>.md` | `python3 workspace.py generate --agent claude` |
| `.claude/rules/delegation.md` | `workspace.py` + agent descriptions | `python3 workspace.py generate --agent claude` |
| `.claude/rules/skills.md` | `workspace.py` (skills=[...] list) | `python3 workspace.py generate --agent claude` |
| `.claude/skill_catalog.json` | Artisyn catalog state | `python3 -c "from skill_sdk_aila.hooks.claude_skill_discovery import generate_claude_skills_catalog; generate_claude_skills_catalog()"` |
| `.claude/commands/generate.md` | `workspace.py` admin commands | `python3 workspace.py generate --agent claude` |
| `.claude/commands/validate.md` | `workspace.py` admin commands | `python3 workspace.py generate --agent claude` |
| `.claude/commands/info.md` | `workspace.py` admin commands | `python3 workspace.py generate --agent claude` |
| `steering-docs/aila-code-doc/<repo>/*/KNOW.md` | AST extraction over cloned repos | `aila-workspace` doc generation (see BOOTSTRAP) |

## Files that CAN be rewritten freely

| File | Editing notes |
|---|---|
| `prompts/*.md` | Source of truth for agent bodies. Edit here, then regenerate. |
| `workspace.py` | Declarative workspace definition. Edit here to add/remove agents, change descriptions, add skills. |
| `specs/*.md` | Not generated. Free to write/edit/delete. Preferred home for scope files. |
| `steering-docs/project-kb/*.md` | Domain KB. Not generated (except through skills). Free to edit. |
| `myproj-feature-area/` (or any project subdirectory) | Free to edit. Not touched by workspace.py. |

## Rewrite-target substitution rules

When the audit's rewrite-proposal step wants to edit a generated file, it
MUST substitute the source-of-truth file:

| Wanted to edit | Substitute with | Additional step |
|---|---|---|
| `CLAUDE.md` | Add content to `prompts/lead.md` or `workspace.py` description | Regenerate |
| `.claude/agents/<role>.md` body | `prompts/<role>.md` | Regenerate |
| `.claude/agents/<role>.md` description | `workspace.py` Agent(..., description=...) | Regenerate |
| `.claude/agents/<role>.md` tools/model | `workspace.py` Agent(..., tools=..., model=...) | Regenerate |
| `.claude/rules/delegation.md` | `workspace.py` + agent descriptions | Regenerate |

## Special case — orchestrator scope-reading instruction

If `GAP-ORCHESTRATOR-SCOPE` fires on `.claude/agents/workspace-myproj-lead.md`,
the audit MUST propose editing `prompts/lead.md` (source of truth), NOT the
generated file. Include the regeneration command in the rewrite proposal's
"How to apply" line.

## Guardrail check in the audit report

Before emitting any `### Proposed rewrite: <path>` block, the audit:

1. Checks if the workspace is aila-SDK (using markers above)
2. Checks if the target path is in the "MUST NOT be rewritten" list
3. If so, substitutes per the rules above
4. Otherwise, proceeds

Every rewrite proposal in an aila-SDK workspace includes this line at the
bottom:

```
**How to apply:** Edit `<source-of-truth-path>`, then run
`python3 workspace.py generate --agent claude` from the workspace root.
Do NOT edit the generated artifact directly.
```

## Non-aila-SDK workspaces

If none of the markers match, treat the workspace as hand-written. All
files are editable. Apply the standard rewrite proposals from Template A–F
without substitution.

## Verifying the constraint in CI (future)

A follow-up enhancement (out of scope for this skill, but useful to note):
add a pre-commit hook that rejects any commit touching generated files in
aila-SDK workspaces. Detect via the auto-generated markers at file head.
