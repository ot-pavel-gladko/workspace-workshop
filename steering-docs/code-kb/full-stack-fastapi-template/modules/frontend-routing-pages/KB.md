# frontend-routing-pages

## Purpose

Implements the full TanStack Router file-based route tree for the frontend. Covers public auth pages (login, signup, recover/reset password) and the authenticated `_layout` shell with its child screens (dashboard, items, settings, admin). `main.tsx` bootstraps the React application by wiring the router, query client, theme provider, and API token management.

## Public Interface

### Routes

| File | URL | Auth required | Renders |
|------|-----|---------------|---------|
| `routes/login.tsx` | `/login` | No | Email/password login form |
| `routes/signup.tsx` | `/signup` | No | Registration form (email, name, password) |
| `routes/recover-password.tsx` | `/recover-password` | No | Email form to trigger password recovery |
| `routes/reset-password.tsx` | `/reset-password?token=…` | No | New-password form; reads token from search params |
| `routes/_layout/index.tsx` | `/` | Yes | Dashboard — welcome greeting |
| `routes/_layout/items.tsx` | `/items` | Yes | Paginated items data table |
| `routes/_layout/settings.tsx` | `/settings` | Yes | Tabbed user settings (profile, password, delete account) |
| `routes/_layout/admin.tsx` | `/admin` | Yes + superuser | User management data table |

### Entry point

`main.tsx` — mounts `<ThemeProvider>` > `<QueryClientProvider>` > `<RouterProvider>` > `<Toaster>`. Also configures `OpenAPI.BASE` from `VITE_API_URL` and attaches a JWT token loader from `localStorage`; a shared `handleApiError` on both `QueryCache` and `MutationCache` clears the token and redirects to `/login` on 401/403.

## Internal Structure

- **`routes/__root.tsx`** — outermost route; renders `<HeadContent>`, `<Outlet>`, React Query Devtools, and TanStack Router Devtools. Provides `NotFound` and `ErrorComponent` fallbacks.
- **`routes/_layout.tsx`** — authenticated shell. `beforeLoad` calls `isLoggedIn()`; redirects to `/login` if false. Renders `SidebarProvider > AppSidebar + SidebarInset(header + Outlet + Footer)`.
- **`routes/_layout/*`** — four child screens rendered inside the layout shell (`index`, `items`, `settings`, `admin`). `admin.tsx` has an additional superuser guard in its own `beforeLoad`.
- **Public pages** (`login`, `signup`, `recover-password`, `reset-password`) — standalone routes outside `_layout`, wrapped in `AuthLayout`.
- **`routeTree.gen.ts`** — auto-generated by TanStack Router CLI; not hand-edited.

## Conventions

- **File-based routing** — directory structure under `frontend/src/routes/` maps 1-to-1 to URL paths; `routeTree.gen.ts` is regenerated automatically.
- **`_layout` prefix** — marks an authenticated shell route; all child routes inherit the auth guard and sidebar layout.
- **Route guards** — implemented via TanStack Router's `beforeLoad` hook; `_layout.tsx` checks `isLoggedIn()`, `admin.tsx` also checks `currentUser.is_superuser`.
- **Forms** — all auth forms use `react-hook-form` with `@hookform/resolvers/zod`; password-match rules use `zod`'s `.refine()`.
- **Error handling** — global 401/403 interception in `main.tsx`; per-route error boundaries via `__root.tsx`.
